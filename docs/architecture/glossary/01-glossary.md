# Enterprise Glossary

A ubiquitous language is critical for Domain Driven Design. This glossary defines the core terminology used across the VeriField Nexus CIOS.

## Core Terminology

- **CIOS (Climate Infrastructure Operating System):** The category of software that VeriField Nexus represents—a foundational operating layer rather than a standalone SaaS app.
- **Organization:** A multi-tenant entity representing a Developer, Regulator, VVB, or Buyer.
- **Jurisdiction:** A legally defined geographical boundary where specific climate policies and NDCs apply.
- **Methodology:** A scientific standard (e.g., AMS-II.G) containing the formulas to calculate carbon impact from field data.
- **PDD (Project Design Document):** The foundational document outlining the scope, methodology, and expected impact of a Project.
- **Asset:** A physical unit deployed in the field (e.g., a single cookstove, a solar home system).
- **Evidence:** Raw data (photos, telemetry, surveys) proving the existence or operation of an Asset.
- **Verification:** The process of an internal QA agent assessing Evidence.
- **Validation / Audit:** The process of an independent third-party (VVB) assessing the Project or Verified Evidence.
- **Carbon Credit:** A digital, ledger-tracked certificate representing 1 metric tonne of CO2 equivalent (tCO2e) reduced or removed.
- **Registry:** An external ledger (e.g., Verra, Gold Standard) that holds the ultimate authority over credit issuance.
- **Retirement:** The permanent removal of a Carbon Credit from circulation so its environmental benefit can be claimed by an end buyer.
